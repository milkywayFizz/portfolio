Web Portfolio (Static)

The goal of this project wasn't just to create a space to showcase my work, but to demonstrate a concept I strongly believe in: the integrity and security of a system start from the foundation.

Today we are used to seeing constant data breaches or unnecessarily heavy websites, often caused by a certain architectural laziness. There's a tendency to import dozens of third-party external libraries without really knowing what they do under the hood. For this portfolio, I chose a different path: a strict Zero-Dependencies philosophy. No heavy frameworks, just semantic HTML, Vanilla JS, and pure CSS3.

This approach not only eliminates trackers and drastically reduces the attack surface, but it also answers a personal principle of mine: things must simply work well for everyone. Whether you're browsing from a high-end workstation or a 400-euro laptop, this site will remain fluid, lightweight, and above all, will fully respect your privacy.

Architecture & Workflow

In life and in projects, I appreciate a very specific mindset: minimum effort, maximum gain. This doesn't mean working less, but working better, optimizing resources. For me, a digital product must simply work, and it must do so at peak performance regardless of the hardware.

To achieve this level of total control, I structured the development into very rigorous phases:

  • Bare-Metal Study: Before touching CSS or complex scripts, I started from the basics. I filled pages of notes studying the native behavior of HTML, the DOM structure, anchors, and space management.
  • AI-Assisted Development: Once the architecture was clear, I integrated Artificial Intelligence with a strict hierarchy: I provided the brains, the AI got its hands dirty. I used it as a true "Tactical Executor". I defined the engineering direction, the logical choices, and the security constraints, delegating the physical writing of the code to it.

Implementing, testing, and manually "dissecting" the generated code allowed me to avoid memorizing anything in a sterile way. On the contrary, it helped me internalize the intimate inner workings of the site. Thanks to this workflow, today I know every millimeter of this project and possess the know-how to expand and modify it with total autonomy.

Engineering Challenges (UI/UX)

Creating an interface that felt "alive" and responsive to the touch, while refusing to use conveniences like animation frameworks (e.g., GSAP), required a lot of creativity and a few headaches.

The first major challenge was text animation. I didn't want the usual static block appearance; I wanted a natural, soft, and fluid flow, which I dubbed the "Lego Effect". To achieve this, I engineered a script that destroys and rebuilds the text DOM in real-time:

javascript 
const splitTextIntoSpans = (selector) => {
                    document.querySelectorAll(selector).forEach(element => {
                    const words = element.innerText.split(' ');
                     element.innerHTML = ''; 
        
                    words.forEach(word => { 
                     const span = document.createElement('span');
                        span.classList.add('stagger-word');
                        span.innerHTML = word + '&nbsp;';
            
                        // THE MAGIC: Injecting an asymmetrical delay into the CSS
                        span.style.setProperty('--random-delay', Math.random());
                        element.appendChild(span);
                        });
                    });
                    };

By injecting a random CSS variable (--random-delay) for every single word, the CSS hardware-accelerates an asymmetrical falling animation. The result is an incredibly organic rain of characters.

Core Snippets: The Carousel Physics Engine

The interactive logo carousel helped me understand the fundamental concepts of hardware optimization. Instead of using pre-packaged sliders (like Swiper or Slick) that would have added dozens of useless Kilobytes of dead weight, I wrote a physics engine from scratch, based on vector mouse tracking.

javascript 
const onDrag = (e) => {
    if (!isDragging) return;
    
    // Hardware fail-safe: forced release if the click event is lost
    if (e.type === 'mousemove' && e.buttons === 0) {
        stopDrag();
        return;
    }
    e.preventDefault(); 
    
    const currentPosition = e.pageX || e.touches[0].pageX;
    const rawDelta = currentPosition - lastX;
    
    // Sensitivity dampening and vector calculation
    const smoothedDelta = rawDelta * dragSensitivity;
    currentX += smoothedDelta;
    
    // Velocity clamping to prevent visual instability
    velocity = Math.max(-maxVelocity, Math.min(maxVelocity, smoothedDelta)); 
    
    lastX = currentPosition;
};

Why is this code block solid from an engineering standpoint?

  • Anti-Sticky Bug: By brute-force reading the hardware state of the mouse (e.buttons === 0), we avoid the annoying glitch where the carousel stays glued to the cursor if the user exits the window while clicking.
  • Rev Limiter: The use of Math.max/min acts as a "disc brake", preventing overly sudden movements from launching the images at impossible-to-render speeds.
  • Zero CPU Bottleneck: This script limits itself to calculating the math of the currentX variable. The actual movement on the screen is delegated to an isolated requestAnimationFrame that uses translate3d, forcing the Graphics Card (GPU) to do the heavy lifting. No lag, a solid 60 frames per second.
