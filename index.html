Web Portfolio (Static)

L'obiettivo di questo progetto non era solo creare uno spazio per esporre i miei lavori, ma dimostrare un concetto in cui credo molto: l'integrità e la sicurezza di un sistema partono dalle basi.

Oggi siamo abituati a vedere continue violazioni di dati o siti inutilmente pesanti, spesso causati da una certa pigrizia architetturale. Si tende a importare decine di librerie esterne di terze parti senza sapere davvero cosa facciano sotto il cofano. Per questo portfolio ho scelto una strada diversa: una rigorosa filosofia Zero-Dependencies. Niente framework pesanti, solo HTML semantico, Vanilla JS e CSS3 puro.

Questo approccio non solo azzera i traccianti e riduce drasticamente la superficie d'attacco, ma risponde a un mio principio personale: le cose devono semplicemente funzionare bene per tutti. Che tu stia navigando da una workstation di fascia alta o da un portatile da 400 euro, questo sito rimarrà fluido, leggero e, soprattutto, rispetterà totalmente la tua privacy.

Architettura & Workflow

Nella vita e nei progetti apprezzo una forma mentis ben precisa: minimo sforzo, massimo guadagno. Questo non significa lavorare meno, ma lavorare meglio, ottimizzando le risorse. Per me un prodotto digitale deve semplicemente funzionare, e deve farlo al massimo delle prestazioni a prescindere dall'hardware.

Per avere questo livello di controllo totale, ho strutturato lo sviluppo in fasi molto rigorose:

  • Studio Bare-Metal: Prima di toccare CSS o script complessi, sono partito dalle basi. Ho riempito appunti studiando il comportamento nativo dell'HTML, la struttura del DOM, le ancore e la gestione degli spazi.
  • Sviluppo AI-Assisted: Una volta chiara l'architettura, ho integrato l'Intelligenza Artificiale con una gerarchia netta: io ci mettevo il cervello, l'IA si sporcava le mani. L'ho utilizzata come un vero e proprio "Tactical Executor". Io definivo la direzione ingegneristica, le scelte logiche e i vincoli di sicurezza, delegando a lei la stesura materiale del codice.

Implementare, testare e "sezionare" manualmente il codice generato mi ha permesso di non dover imparare nulla a memoria in modo sterile. Al contrario, mi ha aiutato a interiorizzare il funzionamento intimo del sito. Grazie a questo workflow, oggi conosco ogni millimetro di questo progetto e possiedo il know-how per espanderlo e modificarlo in totale autonomia.

Sfide Ingegneristiche (UI/UX)

Creare un'interfaccia che sembrasse "viva" e reattiva al tocco, pur rifiutando di usare comodità come i framework di animazione (es. GSAP), ha richiesto molta creatività e qualche mal di testa.

La prima grande sfida è stata l'animazione dei testi. Non volevo la solita comparsa statica a blocchi, volevo un flow naturale, morbido e fluido, che ho ribattezzato "Lego Effect". Per ottenerlo, ho ingegnerizzato uno script che distrugge e ricostruisce il DOM testuale in tempo reale:

javascript 
const splitTextIntoSpans = (selector) => {
                    document.querySelectorAll(selector).forEach(element => {
                    const words = element.innerText.split(' ');
                     element.innerHTML = ''; 
        
                    words.forEach(word => { 
                     const span = document.createElement('span');
                        span.classList.add('stagger-word');
                        span.innerHTML = word + '&nbsp;';
            
                        // LA MAGIA: Iniettiamo un ritardo asimmetrico nel CSS
                        span.style.setProperty('--random-delay', Math.random());
                        element.appendChild(span);
                        });
                    });
                    };

Iniettando una variabile CSS casuale (--random-delay) per ogni singola parola, il CSS accelera via hardware un'animazione di caduta asimmetrica. Il risultato è una pioggia di caratteri incredibilmente organica.

Core Snippets: Il Motore Fisico del Carosello

Il carosello dei loghi interattivo mi ha aiutato a comprendere i concetti fondamentali dell'ottimizzazione hardware. Invece di usare slider pre-confezionati (come Swiper o Slick) che avrebbero aggiunto decine di Kilobyte inutili di zavorra, ho scritto un motore fisico da zero, basato sul tracciamento vettoriale del mouse.

javascript 
const onDrag = (e) => {
    if (!isDragging) return;
    
    // Fail-safe hardware: sgancio forzato se si perde l'evento click
    if (e.type === 'mousemove' && e.buttons === 0) {
        stopDrag();
        return;
    }
    e.preventDefault(); 
    
    const currentPosition = e.pageX || e.touches[0].pageX;
    const rawDelta = currentPosition - lastX;
    
    // Smorzamento della sensibilità e calcolo del vettore
    const smoothedDelta = rawDelta * dragSensitivity;
    currentX += smoothedDelta;
    
    // Clamp della velocità per prevenire instabilità visiva
    velocity = Math.max(-maxVelocity, Math.min(maxVelocity, smoothedDelta)); 
    
    lastX = currentPosition;
};

Perché questo blocco di codice è ingegneristicamente solido?

  • Anti-Sticky Bug: Leggendo brutalmente lo stato hardware del mouse (e.buttons === 0), evitiamo il fastidioso glitch in cui il carosello rimane incollato al cursore se l'utente esce dalla finestra mentre clicca.
  • Limitatore di Giri: L'uso di Math.max/min funge da "freno a disco", impedendo che movimenti troppo bruschi lancino le immagini a velocità impossibili da renderizzare.
  • Zero CPU Bottleneck: Questo script si limita a calcolare la matematica della variabile currentX. Lo spostamento reale a schermo è delegato a un requestAnimationFrame isolato che utilizza translate3d, forzando la Scheda Video (GPU) a svolgere il lavoro pesante. Niente lag, 60 frame al secondo piantati.
